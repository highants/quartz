## Geminiレポート：Obsidian Linterと正規表現で実現する、Markdown記法の自動整形術

### 解説：Markdownでよくある「見た目が崩れる」問題とその原因

皆さんは、Obsidianで書いたMarkdownが、ウェブサイトなどで表示した時に意図しない見た目になってしまった経験はありませんか？

例えば、引用した言葉を強調しようとして `「**重要な言葉**」` と書いたとします。Obsidianのプレビューではきちんと太字に見えているのに、Quartzやブログサービスで公開すると `「**重要な言葉**」` のように、アスタリスク(`*`)がそのまま表示されてしまうことがあります。

これは、MarkdownをHTMLに変換するプログラム（パーサー）の解釈の違いによって起こる現象です。

*   **Obsidianのパーサー**: 比較的柔軟で、多少ルールから外れた書き方でも意図を汲み取ってくれることが多いです。
*   **ウェブサイトのパーサー**: より厳密なルールで解釈するため、ルール違反の書き方はただの文字として扱ってしまうことがあります。

この問題の根本的な原因は、多くのパーサーが `**` のような強調記号を「単語の区切り（スペースなど）」に隣接している場合にのみ有効と判断するルールを持っているためです。`**「` のように記号に直接くっついていると、強調の合図として認識されないことがあるのです。

正しい書き方は、強調したい部分だけを囲む `「**重要な言葉**」` です。これなら、ほとんどの環境で正しく太字として表示されます。

しかし、すでに大量のノートを古い書き方で作成してしまった場合、一つ一つ手で直していくのは大変ですよね。そこで、この修正作業を自動化する方法をご紹介します。

### 解決：Linterプラグインによる一括・自動修正のテクニック

Obsidianのコミュニティプラグイン「**Linter**」を使えば、このような書式の問題を一瞬で、しかも自動で修正することが可能です。ここでは、その具体的な設定方法と、設定に使う「正規表現」という強力なツールの仕組みを解き明かしていきます。

#### 1. Linterのセットアップ手順

まずはLinterプラグインを導入します。

1.  Obsidianの `設定` > `コミュニティプラグイン` を開きます。
2.  `コミュニティプラグインを参照` をクリックし、「Linter」と検索してインストールします。
3.  インストール後、プラグインを有効化してください。
4.  Linterの設定画面を開きます（`設定` > `コミュニティプラグイン` > `Linter`）。

#### 2. 自動修正ルールの追加

Linterの設定項目の中から、「**Custom Regex Replacements**」という項目を探し、以下のルールを新しく追加します。これは「独自のルールでテキストを検索・置換する」機能です。

*   **Find (検索する文字列)**:
    ```
    \*\*「(.*?)」\*\*
    ```
*   **Replace (置換後の文字列)**:
    ```
    「**$1**」
    ```

![](https://cdn.bsky.app/img/feed_fullsize/plain/did:plc:aqrbayxkmfs4tdhcg3c5mb73/bafkreibhocutglsg2yf6zq3qpxt5m652bn7w4wrkqrdrvl2addbq65pw64@jpeg)

これを設定しておくと、Linterを実行（手動実行やファイル保存時に自動実行）するだけで、ノート内にあるすべての `「**文言**」` が、正しい `「**文言**」` 形式に一括で変換されます。

#### 3. 「正規表現」という呪文の解読

さて、先ほど入力した `\*\*「(.*?)」\*\*` や `「**$1**」` は、一見すると意味不明な「呪文」のように見えるかもしれません。これが**正規表現**です。正規表現とは「文字列のパターンを表現するための特殊なルール」のことで、使いこなせるとテキスト処理が劇的に楽になります。

今回の「呪文」を分解して、その意味を解説しましょう。

**検索パターン: `\*\*「(.*?)」\*\*`**

*   `\*\*`
    `*` は正規表現で特別な意味を持つ記号なので、「これはただのアスタリスク文字ですよ」と伝えるために、手前に `\` (バックスラッシュ) を付けています。つまり、`**` という文字列そのものを探します。
*   `「` と `」`
    これは見たまま、`「` と `」` という文字を探します。
*   `(.*?)`
    ここが一番のキモです。3つの記号の組み合わせでできています。
    1.  `.` (ドット): 「改行以外の、なんでもいい一文字」を表すワイルドカードです。
    2.  `*?` (怠惰なアスタリスク): 直前の文字（この場合は `.`）が「0回以上、ただし最短で」繰り返されることを意味します。`「こんにちは」と「さようなら」` という文があった場合、最初の `「` から一番近い `」` まで（`こんにちは`）を見つけてくれます。もし `?` がないと、最後の `」` まで（`こんにちは」と「さようなら`）を一つの塊と認識してしまうので、`?` は非常に重要です。
    3.  `()` (丸括弧): この括弧で囲んだ部分にマッチした文字列を、**後で再利用するために記憶（キャプチャ）しておきます**。今回は `「` と `」` の間にあった「文言」の部分が記憶されます。

**置換パターン: `「**$1**」`**

*   `$1`
    これは特殊な変数で、「**検索パターンで記憶した、1番目の括弧の中身**」を呼び出すための合言葉です。
*   つまり、`「**` と `**」` の間に、先ほど記憶した「文言」を挿入して、`「**文言**」` という文字列を完成させる、という意味になります。

この仕組みによって、Linterはノート内の間違ったパターンを正確に見つけ出し、正しい形へと自動で変換してくれるのです。
